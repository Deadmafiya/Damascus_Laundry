<objective>
## Goal
Stand up the damascus_laundry Cargo workspace so it is **deterministic by construction**:
a `u128` fixed-point math core (no floats in the value path), injectable nondeterministic
dependencies (`Clock`, `Rng`, `Feed`), structured logging, and CI. Every later phase
inherits reproducibility and backtest-live parity for free.

## Purpose
Determinism and fixed-point precision cannot be retrofitted onto a live system (research:
DST lineage + fixed-point precision). Building these foundations first is what makes the
Phase 5 simulation core honest and the Phase 6 golden-file replay possible. This is the
ONE thing v1.0's accuracy depends on.

## Output
A compiling Rust workspace with: a `dl-core` crate (fixed-point + traits), placeholder
crates for the pipeline, property tests passing, `cargo fmt`/`clippy`/`test` green, and a
CI workflow. No network or DEX logic yet.
</objective>

<task type="auto">
  <name>Implement u128 fixed-point math core with property tests</name>
  <files>crates/dl-core/src/fixed.rs, crates/dl-core/src/amount.rs, crates/dl-core/src/lib.rs, crates/dl-core/tests/fixed_props.rs</files>
  <action>
  In `dl-core`, implement the value-path math. No `f32`/`f64` anywhere except clearly
  isolated display/formatting helpers at the boundary.
  - `Amount`: a token amount in smallest base units backed by `u128`, carrying its token
    `decimals` (u8). Provide checked `add`/`sub`/`mul_div` that compute in `u128` (widen to
    `u128` intermediates; if a wider type is needed for a product use explicit 256-bit via
    a small helper or stepwise mul_div) and return `Result<_, MathError>` on overflow —
    never wrap or panic.
  - Decimals normalization: convert an `Amount` to/from a common internal fixed scale and
    back; round-trip must be lossless for representable values (assert in tests).
  - A `mul_div(value, numerator, denominator)` floor-division helper that is the single
    primitive for applying fee fractions and price ratios, overflow-checked.
  - `MathError` enum (Overflow, DivByZero, ScaleMismatch) implementing `std::error::Error`.
  Write `proptest` property tests asserting: (a) `mul_div` never panics on random
  u128 inputs (returns Ok or a typed Err); (b) decimals round-trip is the identity for
  random representable amounts; (c) monotonicity — increasing the numerator of `mul_div`
  never decreases the result; (d) add/sub inverse on non-overflowing pairs.
  </action>
  <verify>`cargo test -p dl-core` passes; `grep -rnE '\b(f32|f64)\b' crates/dl-core/src` returns no matches outside an explicitly named display/format module</verify>
  <done>AC-2 satisfied: u128 fixed-point, overflow-safe, lossless decimals, float-free value path</done>
</task>

<task type="auto">
  <name>Define injectable Clock/Rng/Feed traits with real + deterministic impls</name>
  <files>crates/dl-core/src/clock.rs, crates/dl-core/src/rng.rs, crates/dl-core/src/feed.rs, crates/dl-core/src/lib.rs, crates/dl-core/tests/determinism.rs</files>
  <action>
  Define three traits and two impls each (research: DST — make all nondeterministic
  components pluggable from day one):
  - `Clock`: `now()` / monotonic tick + a `slot()` concept (Solana slots ~400ms). Impls:
    `SystemClock` (real) and `MockClock` (manually/seed-advanced, deterministic).
  - `Rng`: minimal `next_u64()` / bounded helpers. Impls: a real seeded PRNG wrapper and a
    `SeededRng` whose sequence is fully determined by its seed. (Do NOT use system entropy
    in the deterministic path.)
  - `Feed`: an async or iterator-style source yielding `FeedEvent` items (define a minimal
    `FeedEvent` enum now — e.g. `AccountUpdate{ slot, pubkey, data }`, `Slot{ slot }` — to
    be extended in Phase 2). Impls: a `NullFeed`/`ScriptedFeed` that replays a fixed,
    in-memory list of events deterministically. (Real network feed is Phase 2.)
  Keep the traits object-safe / generic so `dl-app` can wire either impl. Add a
  `determinism.rs` integration test: run a tiny driver loop twice with `MockClock` +
  `SeededRng` + `ScriptedFeed` on the same seed/script and assert the two recorded output
  sequences are byte-for-byte equal.
  </action>
  <verify>`cargo test -p dl-core --test determinism` passes (two seeded runs produce identical output)</verify>
  <done>AC-3 satisfied: injectable nondeterministic deps reproducible under seed</done>
</task>
