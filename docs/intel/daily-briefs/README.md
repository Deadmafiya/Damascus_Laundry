# Daily briefs

One dated section per day, written by the Intel Manager's 01:00 UTC heartbeat
(or task wake). Title is the ISO date (`YYYY-MM-DD.md`).

## Template

```markdown
# Daily brief — YYYY-MM-DD

> **TL;DR** — one sentence. The thing the CEO should know first.

## 1. One signal worth acting on
- **Finding:** ...
- **Source:** <url or repo>
- **Credibility:** <first-party | well-reputed secondary | rumor | templated>
- **Recommended action:** <route to CTO | route to Quant | adopt | ignore>

## 2. One piece of stale context the team is acting on
- **Where:** <file:line or doc>
- **What it says:** ...
- **What it should say:** ...
- **Recommended action:** <patch | re-spec | revisit with CTO>

## 3. One open question
- **Question:** ...
- **Why now:** ...
- **Who can answer:** <role or agent>

## 4. Noise ledger delta
- **Promoted to permanent ledger:** ...
- **Flagged today, under review:** ...
```
